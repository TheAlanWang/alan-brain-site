### Inheritance and Abstract Classes
implements and extended
AbstractShape.java

| Feature      | `extends` (Inheritance)                           | `implements` (Interface)                      |
| ------------ | ------------------------------------------------- | --------------------------------------------- |
| Purpose      | A class inherits another class/**abstract class** | A class implements an interface               |
| Code reuse   | Gains the parent class’s existing implementation  | Must implement all interface methods yourself |
| Limitation   | Can only extend **one** class                     | Can implement **multiple** interfaces         |
| Relationship | “is-a” (Dog is an Animal)                         | “can-do” (Duck can fly)                       |
**shape is an interface**
extend AbstractShape -> implement shape

```java
abstract class Shape {
    abstract double area();
}

interface ComparableShape extends Comparable<Shape> {}

class Circle extends Shape implements ComparableShape {
    private double r;

    Circle(double r) { this.r = r; }

    @Override
    double area() {
        return Math.PI * r * r;
    }

    @Override
    public int compareTo(Shape other) {
        return Double.compare(this.area(), other.area());
    }
}

```

### Access modifiers
- **`private`**: can only be accessed within the same class.
- **`public`**: can be accessed everywhere.
- **`protected`**: can be accessed by the class itself and its subclasses (but not directly by external classes).

### Composition, Abstraction, Inheritance
