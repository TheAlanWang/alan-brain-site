### MVC
- Q: Why need mvc?
- A: Use MVC as a system design pattern, to reduce the code complexity by separate the program into three parts.

- Model: 
	- back-end design
	- does not care about which functionality is used when, how results are shown to the user
- View: 
	- Display to the user
	- does not care how the results were produced, when to respond to user action
- Controller: 
	- Accepts user input
	- Translates user actions into operation
