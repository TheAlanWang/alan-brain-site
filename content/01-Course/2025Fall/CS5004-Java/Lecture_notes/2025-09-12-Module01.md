## Four Key OOP Characteristics in Java
- **Encapsulation**
	- Bundling data and methods inside a class and hiding implementation
- **Inheritance**: 
	- Subclasses reuse the properties and methods of a superclass, forming a hierarchy.
- **Polymorphism**: 
	- The same method call can behave differently on different objects.
- **Abstraction**: 
	- Defining what to do without specifying how to do it (via abstract classes or interfaces).
## First Java Code: Hello World
```java
public class HelloWorld {  
  public static void main(String[] args){  
    System.out.println("Hello, world\n");  
  }  
}
```

- `static`:  Belongs to the class, not an object. You can run `main` without creating an object.
- `main` : The special **entry point method** in Java. The JVM always looks for `main`.
- `(String[] args)`: A parameter, an **array of Strings**, containing command-line arguments.

## First Java Class
### class contains

Person.java
```java
/**  
 * This class represents a person with a first name, last name and a * year of birth */
public class Person { 
  private String firstName;  
  private String lastName;  
  private int yearOfBirth;  
   
  /**  
   * Constructs a Person object and initializes it to the given first name, last   * name and year of birth   ** @param firstName   the first name of this person  
   * @param lastName    the last name of this person  
   * @param yearOfBirth the year of birth of this person  
   */  
   public Person(String firstName, String lastName, int yearOfBirth) {  
    this.firstName = firstName;  
    this.lastName = lastName;  
    this.yearOfBirth = yearOfBirth;  
  }  
   
  /**  
   * Gitter
   */  
   public String getFirstName() {  
    return this.firstName;  
  }  
      
  /**  
   * Return the age of this person in 2020   ** @return the age of this person in 2020  
   */  
   public int getAge() {  
    return 2020 - this.yearOfBirth;  
  }  
   
  /**  
   * Return the full name of this person   ** @return the full name of this person  
   */  
   public String getFullName() {  
    return this.firstName + " " + this.lastName;  
  }  
}
```

## JAVA strict rules
- **One `public` class per file**
- `public class` in Java, the filename must match the class name **exactly** (case-sensitive).
- Main method signature `public static void main(String[] args)`
- Package structure must match folder structure
- No two top-level classes with the same name in one package
- Access modifiers rules
- Type safety rules
